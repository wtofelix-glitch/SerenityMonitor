# 权重调优分析报告

## 📋 护城河（moat）维度在评分系统中的角色分析

### 1.1 护城河 vs 动量 vs 情绪 — 三者的本质差异

| 维度 | 衡量什么 | 时间跨度 | 数据来源 | 与市场相关度 |
|------|----------|----------|----------|-------------|
| **护城河 (moat)** | 企业结构性竞争优势（ROIC稳定性、毛利率趋势、负债安全、市场地位、资本效率） | **3-5年** | 财报基本面 | **低** — 独立于市场情绪 |
| **动量 (momentum)** | 短期价格趋势强度 | **5-20个交易日** | 价格K线 | **高** — 跟随市场 |
| **情绪 (sentiment)** | 新闻/社交媒体的市场情绪 | **1-5个交易日** | 新闻文本 | **高** — 影响市场 |
| **成交量 (volume)** | 交易活跃度/资金流动 | **1-20个交易日** | 交易数据 | **中** — 验证价格 |

**结论：护城河是系统中唯一的长期结构性维度。** 它与动量/成交量/情绪的天然低相关性正是最大的价值所在——它提供了**完全正交的互补信息**。

### 1.2 当前评分系统的信息互补性分析

| 维度对 | 预计相关性 | 互补价值 |
|--------|-----------|---------|
| moat ↔ momentum | **低 (~0~0.2)** | ✅ 高互补 — 高护城河低动量的标的正是逆向投资机会 |
| moat ↔ sentiment | **低 (~0~0.1)** | ✅ 高互补 — 负面情绪时高护城河抗跌 |
| moat ↔ volume | **低 (~0~0.15)** | ✅ 中互补 — 成交量萎缩时护城河作为质量锚 |
| momentum ↔ sentiment | **高 (~0.5~0.6)** | ❌ 信息重叠 — 两者均受市场情绪驱动 |

**关键洞察：** 在 SerenityMonitor 的防御组合（Tier 4：长江电力、工商银行、招商银行）中，这些股票的 momentum 在非防御行情下可能较低，但其 moat 评分却是最高的（长江电力73.1，工商银行65.0）。这恰恰证明了 moat 的"反脆弱"价值——当动量反转时，高护城河的防御标的成为资金避风港。

## 2️⃣ 当前权重分析

### 2.1 权重现状

| 维度 | 默认权重 | 当前调整后权重 | 变化 |
|------|---------|--------------|------|
| base | 0.14 | **0.1682** | 🟢+2.82% |
| zone | 0.14 | **0.1412** | ⚪ +0.12% |
| momentum | 0.14 | **0.1203** | 🔴 -1.97% |
| volume | 0.04 | **0.0406** | ⚪ +0.06% |
| serenity | 0.14 | **0.1300** | 🔴 -1.00% |
| factor | 0.14 | **0.1526** | 🟢+1.26% |
| technical | 0.09 | **0.0902** | ⚪ +0.02% |
| sentiment | 0.09 | **0.0903** | ⚪ +0.03% |
| **moat** | **0.08** | **0.0666** | 🔴 **-1.34%** |

### 2.2 重要发现：当前 0.0666 是数据缺失的副产品

查看 `adjusted_weights.json` 的 `source_ic` 字段，**moat_score 完全不在 IC 数据中**：

```json
"n_days": {
    "total_score": 10, "base_score": 10, "zone_score": 10,
    "momentum_score": 10, "volume_score": 10, "serenity_score": 10,
    "factor_score": 10, "technical_score": 10
    // ❌ moat_score 不存在！
}
```

**根本原因：** `factor_ic.py` 尚未更新以追踪 moat 维度的 Rank IC。当 `weight_adjuster.py` 找不到 moat 的 IC 数据时，它执行了这段逻辑：

```python
if n_days.get(ic_dim, 0) < min_days:
    adjustment[weight_key] = DEFAULT_WEIGHTS[weight_key]  # 保持 0.08
```

所以 moat 的原始调整值保持了 0.08，但在后续 **归一化步骤**中，其他8个维度因 IC 上调而下调了权重，moat 的 0.08 被动地被挤压缩至 **0.0666**。

**这不是一个经过优化的权重，而是一个统计上的"剩余分配"产物。**

### 2.3 与其他维度的对比

| 对比维度 | 权重 | 子维度数 | 信息性质 | 当前 moat vs 该维度 |
|---------|------|---------|---------|-------------------|
| base | 16.82% | 4-5个 | 基本面适应度 | moat 仅为其 **40%** |
| factor | 15.26% | 14因子 | 多因子量化 | moat 仅为其 **44%** |
| momentum | 12.03% | 3-4个 | 价格趋势 | moat 仅为其 **55%** |
| sentiment | 9.03% | 1-2个 | 新闻情绪 | sentiment > moat |
| **volume** | **4.06%** | 2-3个 | 成交量信号 | moat = 1.6× volume |

**问题：** 一个只有2-3个子维度的 volume（成交量检测），其权重为 4.06%；而一个有 5 个子维度、涵盖完整巴菲特量化框架的 moat，权重仅为 6.66%。这种差距不合理。

## 3️⃣ 权重建议

### 3.1 建议：moat 从 0.0666 → **0.10（10%）**

#### 理由：

**① 互补信息价值（核心论据）**
- Moat 与 momentum/volume/sentiment 天然低相关
- 在 9 维评分系统中，**捕获独特方差**的维度应当获得更高权重
- 参考多因子量化理论：低相关因子的配权应高于其IC单独决定的水平

**② 维度复杂度对等**
- Moat 有 5 个子维度（ROIC稳定性、毛利率趋势、负债安全、市场地位、资本效率），每个子维度都有独立的计算逻辑
- 相比之下 volume 只有 2-3 个简单信号（突增/萎缩）却仅略低于 moat
- 为 5 子维度的长周期因子配 10% 权重是合理的下限

**③ 实际影响测算**

|------|---------|------------------------|----------------------|------|
| 600900 长江电力 | 73.1 | **4.87 分** | **7.31 分** | +2.44 |
| 002281 光迅科技 | 69.7 | 4.64 分 | 6.97 分 | +2.33 |
| 600171 上海贝岭 | 68.0 | 4.53 分 | 6.80 分 | +2.27 |
| 600487 亨通光电 | 66.5 | 4.43 分 | 6.65 分 | +2.22 |
| 601398 工商银行 | 65.0 | 4.33 分 | 6.50 分 | +2.17 |
| 600036 招商银行 | 62.7 | 4.18 分 | 6.27 分 | +2.09 |

- 影响范围：**2-2.5 分** — 足以在评分边界附近（如 BUY 阈值 72）左右决策
- 但不会主导总分（不像 base/zone/factor 等更高权重维度）

**④ 不采用更高权重的原因（10% 是上限）**
- Moat 数据依赖财报，更新频率低（季度），在快速变化的市场中反应滞后
- 数据质量受 akshare/估算方案影响，部分股票使用降级估算数据
- 作为参考：主流量化系统中单个基本面因子的权重通常在 5-10%

## 4️⃣ 子维度权重分析

### 4.1 当前5子维度权重

| 子维度 | 当前权重 | 巴菲特框架优先级 | 评分 |
|--------|---------|----------------|------|
| ROIC 稳定性 | **25%** | ★★★★★ 最高 | ✅ 合理偏高 |
| 毛利率趋势 | **20%** | ★★★★ 高 | ✅ 合理 |
| 负债安全 | **20%** | ★★★★ 高（行业敏感） | ✅ 合理 |
| 市场地位 | **15%** | ★★★★ 高 | ❌ 偏低 |
| 资本效率 | **20%** | ★★★ 中高 | ⚠️ 可微调 |

### 4.2 具体分析

#### 🔴 ROIC 稳定性 25% → 建议 22-23%

**问题：**
- ROIC 是**后视镜指标** — 反映过去5年的平均值+波动，但不能预测未来能力
- A 股上市公司财务报表质量参差不齐，akshare 数据有时缺失或不可靠
- 当前25%的权重意味着 ROIC 单维度决定 moat 总分的1/4，对于数据质量不稳定的维度来说偏高

**建议：** 从25%下调至 **22-23%**

#### 🟢 市场地位 15% → 建议 18-20%

**问题：**
- **市场地位是最具前瞻性的子维度** — 行业龙头/寡头地位意味着定价权、客户粘性、规模效应
- 当前15%是唯一低于20%的维度，与巴菲特框架不符
- `_calc_market_position()` 函数依赖 STOCK_MAP 中的 Tier 来判断，数据确定性最高（无财务数据依赖）
- 护城河的核心是"企业能否长期保持竞争优势"，市场地位是最直接的衡量

**建议：** 从15%上调至 **18-20%**

在测试中（以 600487 亨通光电为例）：
- 当前权重：80×0.25 + 50×0.20 + 80×0.20 + 70×0.15 + 50×0.20 = **66.5** ✓
- 建议权重：80×0.22 + 50×0.20 + 80×0.20 + 70×0.20 + 50×0.18 = **66.6** ✓
- 总分几乎不变，但各股票间的相对排名会因 profile 不同而变化

#### ⚪ 毛利率趋势 20% → 保持 20%

毛利率趋势是领先指标（先行于 ROIC 变化），维持现有权重合理。

#### ⚪ 负债安全 20% → 保持 20%

行业敏感设计已经合理区分银行（600036/601398）与普通行业，权重适中。

#### ⚪ 资本效率 20% → 建议 18%

FCF/营收比数据在 akshare 中获取不稳定（经常缺失资本支出数据），导致降级到 OCF 近似。可小幅下调至 **18%**。

## 5️⃣ 总结与行动清单

### 5.1 核心结论

| 项目 | 结论 |
|------|------|
| 当前 moat 权重 0.0666 | ❌ **不合理** — 是 IC 数据缺失导致的剩余分配产物 |
| 建议 moat 权重 | ✅ **0.10（10%）** — 基于互补信息价值 + 维度复杂度 |
| 子维度 ROIC 25% | ⚠️ **建议下调至22%** — 后视镜 + 数据质量 |
| 子维度 市场地位 15% | 🟢 **建议上调至20%** — 最前瞻、最确定性高 |
| 子维度 毛利率 20% | ✅ 保持 |
| 子维度 负债安全 20% | ✅ 保持 |
| 子维度 资本效率 20% | ⚠️ **建议下调至18%** — 数据不稳定 |

### 5.2 行动项

1. **立即修复：** 更新 `factor_ic.py` 以追踪 `moat_score` 的 Rank IC，让动态调整系统能正常工作
2. **手动设置：** 在修复前，将 `DEFAULT_WEIGHTS["moat"]` 从 `0.08` 改为 `0.10`，并重新运行 `weight_adjuster.py --reset` 后重新调整
3. **子维度更新：** 在 `moat_factor.py` 中更新 `MOAT_WEIGHTS` 字典，应用新的子维度权重
4. **监控：** 在新权重运行2-4周后，检查 moat 维度的 IC 贡献，验证权重设定是否合理

### 5.3 风险提示

- Moat 权重从 6.66% 提升至 10% 会增加评分系统对财报数据的敏感度，在财报季可能会有评分跳变
- `market_position` 子维度目前依赖 STOCK_MAP 的 Tier 配置（手工判定），提升其权重意味着需要更准确地维护 Tier 分类
- 防御组合（Tier 4）的评分将小幅提升，可能使买入信号更倾向于防御型标的
